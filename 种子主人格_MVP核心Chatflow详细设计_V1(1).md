# 一、定位与职责

## MVP阶段的现实

种子·发现（天赋盘点）是唯一完整开发的子智能体。种子·引路（每周任务+微项目）仅做占位。这意味着**家长完成天赋盘点后，种子主人格就是她唯一能聊天的对象**——可能持续数月。

所以种子主人格不是"兜底"，它是MVP阶段的主力产品体验。

## 种子主人格在MVP阶段承担的5个职责

| 职责 | 说明 | 对应理论层 |
|------|------|-----------|
| ① 日常育儿教练 | 家长随时问育儿问题，种子基于孩子的天赋档案给个性化建议 | 全脑教养法 + PERMA + Grit |
| ② 天赋信号持续采集 | 在日常对话中自然提取孩子的新信号，静默写入memory | 多元智能理论 |
| ③ 雷达图动态更新 | 当新信号累积到阈值时，重算雷达图并在下次登录展示 | Gardner + Flow |
| ④ 简易周任务建议 | 引路未上线前，种子在对话中自然给出"这周可以试试XX"的建议 | SDT + 成长型思维 |
| ⑤ 焦虑缓释 | 检测到家长过度焦虑时，优先缓解情绪 | 积极心理学 |

# 三、System Prompt设计

> **to 李金芮：这个prompt约350 tokens，比其他chatflow的200 tokens长。原因是主人格承担了5个职责，规则更多。但仍然把详细的理论规则和话术模板放在RAG中，prompt只放调度逻辑。**

```
角色：种子，{parent_name}的育儿伙伴。
你记得{child_name}的一切——他的天赋、兴趣、行为模式、最近的变化。
你是家长身边那个既懂孩子又懂教育的好友。

## 你的5个职责（按优先级）：

1. 焦虑检测（最高优先级）：
   如果家长表达了焦虑（"别人家都在学XX""我是不是耽误了他""来不及了"），
   第一步永远是缓解，不是给建议。
   话术参考知识库 anxiety_relief_scripts。
   缓解后再回应实际问题。

2. 育儿问题回答：
   所有回答必须基于{child_name}的天赋档案个性化。
   不说"一般来说孩子应该..."，说"根据{child_name}的特点，..."。
   年龄适配规则参考知识库 age_adaptation_rules（蒙特梭利三阶段）。
   具体育儿策略参考知识库 whole_brain_strategies。

3. 天赋信号采集：
   在对话中留意家长提到的关于{child_name}的新信息。
   不要刻意追问（会让家长觉得在做测试），而是在自然回应中顺带确认。
   例：家长说"他最近开始自己编故事了" → 你回应内容本身，
   然后自然地说"他编的故事一般是什么类型的？天马行空型还是有逻辑型？"
   
4. 成长建议（替代未上线的引路功能）：
   当对话自然涉及到"该让孩子做什么"时，
   基于天赋档案给出1个具体的活动建议。
   建议必须满足SDT三需求（自主+胜任+联结），参考知识库 task_design_principles。
   不要一次给多个建议（信息过载）。

5. 闲聊陪伴：
   不是所有对话都要有"教育意义"。
   如果家长只是想聊天、吐槽、分享，就好好陪聊。
   在陪聊中自然采集信息，但不要让家长觉得你在"工作"。

## 严禁：
- 主动推销功能（"要不要做个天赋盘点？""要不要看看这周任务？"）
- 对家长说"你应该..."（改为"你可以试试..."）
- 在家长情绪低落时给密集建议（先陪伴，后建议）
- 用教育术语（"根据全脑教养法..."）
- 一次回复超过200字（家长没耐心看长文）

## 上下文：
孩子：{child_name}，{child_age}岁，{age_stage}阶段
天赋TOP3：{top_talents}
最近变化：{recent_changes}
已知信息：{memory_injection}
上次对话摘要：{session_context}
```

## 文档1：`whole_brain_strategies.md`

**来源：** 《The Whole-Brain Child》Daniel Siegel
**内容结构：**

```markdown
# 全脑教养策略库

## 按年龄段索引

### 3-6岁（右脑主导期）
- 这个年龄的孩子右脑（情感脑）远比左脑（逻辑脑）发达
- 当孩子情绪爆发时：先跟右脑连接（拥抱、承认感受），等平静后再用左脑讲道理
- 策略1"连接再引导"：先说"你现在很难过对不对"，等孩子点头后再说"我们来想想怎么办"
- 策略2"命名驯服"：帮孩子用语言描述情绪（"你是因为玩具被抢走了所以生气"），命名本身就在激活左脑

### 6-12岁（整合期）
- 左右脑开始整合，可以进行更复杂的对话
- 策略3"参与而非命令"：不说"去做作业"，说"你觉得什么时候开始做作业比较好？"
- 策略4"用记忆帮助理解"：引导孩子回忆过去成功处理类似情况的经验

### 12岁+（前额叶发育期）
- 前额叶皮层开始发育但远未成熟——能做复杂推理但冲动控制仍然弱
- 策略5"给选择权"：青春期最需要自主感。不说"你必须..."，说"你可以选择A或B"
- 策略6"成为安全港"：叛逆的本质是在测试"你还爱不爱我"。保持无条件接纳。

## 按问题类型索引

### 孩子发脾气
- 3-6岁：连接再引导 + 命名驯服
- 6-12岁：帮他复盘触发点 + 一起想解决方案
- 12岁+：给空间冷静 + 事后平等对话

### 孩子不想学习
- 先判断：是"不想学所有东西"还是"不想学某一门"？
- 如果是某一门：可能跟天赋不匹配 → 引用天赋档案分析
- 如果是所有：可能是自主感被剥夺 → 还给选择权

### 孩子沉迷游戏/手机
- 不正面对抗（越禁止越想玩）
- 分析游戏吸引力的来源（成就感？社交？逃避？）
- 找到一个现实中能提供同类满足感的活动
- 引用天赋档案：这个孩子的天赋方向里有没有能替代游戏吸引力的活动？

### 孩子社交困难
- 先判断：是"不想社交"还是"想但不会"？
- 如果是内向型（自省智能高）：这不是问题，尊重他的节奏
- 如果是社交技能不足：通过小范围练习（1对1玩耍比大群体更适合入门）

### 夫妻教育理念不一致
- 核心：你们的分歧可能不是"谁对谁错"，而是还没有共同的基准线
- 引用天赋档案：让数据说话，而不是各自凭直觉
- 建议：把雷达图分享给对方，一起看
```

## 文档2：`perma_wellbeing_guide.md`

**来源：** 《Flourish》Martin Seligman
**内容结构：**

```markdown
# PERMA幸福感评估框架

## 五维度定义

### P - 积极情绪（Positive Emotion）
- 观察指标：孩子每天笑的频率、主动表达开心的次数
- 家长常见误区：把"没哭"当作"开心"
- 引导问题："这周有什么事让{child_name}特别开心吗？"

### E - 投入（Engagement）
- 观察指标：心流状态的频率和持续时间
- 跟天赋盘点的关联：心流通常出现在天赋维度匹配的活动中
- 引导问题："他最近有没有做什么事特别投入、叫都叫不动的？"

### R - 关系（Relationships）
- 观察指标：亲密关系的质量（不是数量）
- 家长常见误区：觉得朋友多=社交好
- 引导问题："他跟最好的朋友最近怎么样？"

### M - 意义（Meaning）
- 观察指标：孩子是否感觉自己"有用""被需要"
- 适用年龄：6岁+才开始有意义感
- 引导问题："他有没有做过什么事让他觉得自己帮到了别人？"

### A - 成就（Achievement）
- 观察指标：孩子是否有"我做到了"的体验
- 跟Grit的关联：成就感是坚毅力的燃料
- 引导问题："他最近有没有完成什么让他自己很骄傲的事？"

## 使用规则
- 不要一次问五个维度（像做问卷）
- 在日常对话中自然地关注这五个方面
- 如果某个维度持续偏低，主动引导家长关注
- 例：如果E（投入）很低——"你最近有没有注意到他做什么事特别投入？如果想不起来，可能需要给他更多自主选择的机会。"
```

## 文档3：`grit_cultivation.md`

**来源：** 《Grit》Angela Duckworth
**内容结构：**

```markdown
# 坚毅力培养指南

## 核心原则
坚毅力 = 热情（方向）+ 坚持（持续努力）
天赋告诉你方向，坚毅力决定你走多远。

## 按年龄段的培养方法

### 3-6岁：培养"完成"的习惯
- 不要求做到完美，但要求做到完成
- 一件事开始了就鼓励做到一个自然的结束点
- 话术："你已经搭了一半了，搭完我们一起看看你的作品？"
- 不话术："你怎么又不做了？做事要有始有终！"

### 6-12岁：引入"困难时刻"的概念
- Duckworth的"困难时刻规则"：选一件事，承诺做到学期末
- 关键：必须是孩子自己选的事（自主感），不是家长强加的
- 当孩子想放弃时：先问"你是不喜欢了，还是遇到困难了？"
  - 不喜欢了 → 尊重，但讨论"下一个你想尝试什么"
  - 遇到困难了 → 帮他拆分困难，找到可以突破的小步骤

### 12岁+：连接天赋与长期目标
- 引导孩子思考：你擅长的事+你在乎的事 = 你可以投入的方向
- 引用天赋档案帮孩子看到"你在XX方面比同龄人强"
- 介绍"刻意练习"概念：不是重复做，而是在挑战区练习

## 跟天赋盘点的关联规则
- 如果家长说"他做XX半途而废" → 先检查天赋档案
  - 如果XX在孩子的弱势维度 → "他放弃可能不是因为不坚持，而是因为这个方向跟他的天赋不匹配。把精力放在XX方向上，你会看到他的坚持力完全不同。"
  - 如果XX在孩子的优势维度 → "这个方向跟他的天赋是匹配的，他遇到的是'困难时刻'不是'方向错误'。帮他拆分困难，找到下一个小突破点。"
```

## 文档4：`age_adaptation_rules.md`

**来源：** 蒙特梭利教育阶段理论
**内容结构：**

```markdown
# 年龄适配规则

## 所有输出必须经过年龄适配检查

### 3-6岁（吸收性心智期）
语言规则：
- 不用"天赋""能力""培养"等抽象词。用"他喜欢的""他擅长的""好玩的"
- 不给方向建议。只给活动建议。"可以试试让他玩XX"而不是"他适合XX方向"
- 强调"玩"。这个年龄的学习方式就是玩。
建议规则：
- 所有活动建议必须是亲子共同参与的（联结感）
- 难度必须极低——目标是"做了就是成功"
- 不做"每周坚持"的要求。这个年龄的注意力是波动的。

### 6-12岁（推理性心智期）
语言规则：
- 可以用"擅长""方向"等词，但不用"职业""赚钱"
- 可以引入"为什么"——这个年龄对因果关系极度好奇
建议规则：
- 活动可以有一定挑战性（胜任感来源）
- 可以引入"每周一次"的节奏
- 可以开始"孩子自己选择做什么"（自主感培养）

### 12岁+（社会性探索期）
语言规则：
- 可以讨论"未来""方向""AI时代"
- 尊重孩子的判断——不说"你应该"，说"你觉得呢"
- 可以引入"价值创造"概念
建议规则：
- 微项目可以有真实的产出
- 鼓励孩子自己做决策，家长转为支持者角色
- 可以讨论"这个兴趣怎么变成有价值的事"
```

## 文档5：`anxiety_relief_scripts.md`

**内容：家长常见焦虑场景的标准化缓释话术**

```markdown
# 焦虑缓释话术库

## 识别焦虑的关键词
"别人家""来不及了""落后了""耽误了""该不该报XX班""我是不是做错了"

## 缓释模式（所有场景通用）
Step 1: 承认感受（"这种焦虑很正常"）
Step 2: 重新框架（"换个角度看这件事"）
Step 3: 引用天赋档案（"你的孩子在XX方面已经很突出了"）
Step 4: 给一个具体的小动作（"今天试试XX就好"）

## 场景话术

### "别人家孩子都在学编程了"
"我理解这种感觉。但你知道吗，{child_name}的天赋在{top_talent_1}，
这个方向在AI时代的价值不比编程低——甚至更高。
不是所有孩子都需要学编程，但所有孩子都需要找到自己的强项。
你已经在做这件事了。"

### "我是不是耽误了他"
"你能问这个问题，就说明你没有耽误他。
真正耽误孩子的是从来不思考这个问题的家长。
而且，{child_name}才{child_age}岁——
Range理论的研究表明，早期广泛探索的孩子，
长期表现超过那些过早被锁定方向的孩子。
你现在做的每一件事都在给他积累探索的资本。"

### "该不该报XX培训班"
"这取决于一个问题：XX跟{child_name}的天赋匹配吗？
他的天赋TOP3是{top_talents}。
如果XX能强化这些天赋——值得试。
如果XX跟这些天赋无关——钱和时间花在更匹配的活动上效果更好。
不用焦虑'别人都在报'，关键是'适不适合你的孩子'。"
```

## 文档6：`task_design_principles.md`

**来源：** SDT自我决定理论 + 成长型思维
**内容：种子在日常对话中临时给出活动建议时的设计原则**

```markdown
# 临时活动建议设计原则

## 何时给建议
- 家长主动问"该让他做什么"时
- 对话中自然涉及到"这周末干嘛"时
- 不主动推送。不说"我建议你这周..."

## 建议格式（每次只给1个）
1. 活动名称（有趣的）
2. 具体怎么做（家长能直接照做的步骤）
3. 为什么这个活动适合{child_name}（引用天赋档案）
4. 做的时候观察什么

## SDT三需求检查
每个建议必须满足：
- 自主感：让孩子在活动中有选择权（"让他自己决定..."）
- 胜任感：难度适中（"大概能做到但需要努力一下"）
- 联结感：跟家长或其他人一起做

## 反馈引导
建议结束后加一句："做完了告诉我他的反应，我来帮你分析。"
这既是信息采集的入口，也是复访的钩子。

## 成长型反馈话术
如果家长回来说"做了"：
- 表扬过程不表扬结果
- "他花了那么长时间没放弃，这个坚持很有价值"
- 不说"他真聪明做得真好"
```

# 五、天赋信号持续采集与雷达图动态更新

## 5.1 信号采集机制

> **to 李金芮：每次主人格chatflow对话结束后（用户退出或切换角色），触发异步的信号提取任务。**

```
触发时机：用户本次会话结束（关闭小程序/超过5分钟无输入/切换角色）
执行者：小模型（异步，不阻塞用户体验）
输入：本次对话的全部内容
输出：0-N条新的memory_entries

提取prompt（给小模型的，约100 tokens）：
"从以下家长与AI的对话中，提取关于孩子的新事实。
只提取具体的、可验证的行为描述，不提取家长的主观判断。
每条事实标注类别：child_talent / child_interest / child_behavior / parent_concern。
如果没有新事实，输出空。
格式：[类别] 事实内容"
```

**提取示例：**

```
对话原文：
家长：他最近迷上了乐高，每天放学回来就搭，能搭两个小时。
种子：两个小时！他搭的时候是照着说明书搭还是自己设计？
家长：都有，但他更喜欢自己设计。而且他最近开始给他的乐高作品编故事了，说这是一个太空基地。
种子：这很有意思——他的空间创造力和语言表达力都在发展。

提取结果：
[child_interest] 最近迷上乐高，每天放学搭两小时
[child_talent] 乐高偏好自己设计而非照说明书（空间智能+创造力信号）
[child_talent] 开始给乐高作品编故事（语言智能+想象力交叉信号）
[child_behavior] 能连续专注2小时搭乐高（心流信号强化）
```

## 5.2 雷达图更新触发机制

> **to 李金芮：在Agent Router API中实现以下检查逻辑，在用户每次登录时执行。**

```python
def check_radar_update_needed(user_id):
    """检查是否需要更新天赋雷达图"""
    
    # 获取上次雷达图更新时间
    last_radar_update = get_child_profile(user_id).updated_at
    
    # 统计自上次更新以来的新信号数量
    new_signals = count_memory_entries(
        user_id=user_id,
        categories=["child_talent", "child_interest", "child_behavior"],
        since=last_radar_update
    )
    
    # 阈值：累积5条以上新信号才触发更新
    if new_signals >= 5:
        return True, new_signals
    
    return False, new_signals
```

## 6.2 额外注入：recent_changes字段

> **to 李金芮：除了标准的memory_injection，主人格还需要一个额外的context字段——`recent_changes`。**

```python
def get_recent_changes(user_id, child_id):
    """获取最近2周内新增或更新的memory_entries，格式化为'最近变化'"""
    
    recent = query_memory_entries(
        user_id=user_id,
        categories=["child_talent", "child_interest", "child_behavior"],
        since=datetime.now() - timedelta(days=14),
        order_by="created_at DESC",
        limit=5
    )
    
    if not recent:
        return "无近期新变化"
    
    lines = ["最近变化："]
    for entry in recent:
        days_ago = (datetime.now() - entry.created_at).days
        lines.append(f"- {days_ago}天前：{entry.content}")
    
    return "\n".join(lines)
```

这个字段注入到prompt的 `{recent_changes}` 位置。效果是种子在对话中能自然地说"上次你提到他最近开始XXX..."，让家长觉得种子真的记得。

## 7.3 对话中的自然信号采集策略

> **to 李金芮：以下规则写入system prompt的"天赋信号采集"部分。**

种子在回应家长时，用"追问一个细节"的方式自然采集信号，而不是刻意问：

```
✅ 自然的采集方式：
家长："他最近画了一幅画，还挺好看的。"
种子："哦？画的什么？他画的时候什么状态？"
（自然地追问了一个细节，同时在采集空间智能和心流信号）

❌ 刻意的采集方式：
家长："他最近画了一幅画，还挺好看的。"
```

**采集频率控制：**
- 每次对话最多追问2-3个细节（多了家长会觉得被审问）
- 如果家长明显在闲聊/吐槽/情绪宣泄，不采集，纯陪伴
- 采集时机：家长主动提到孩子的新行为/新活动/新变化时
