# 一树 OPC —— 服务器怎么选

> 一句话:**4/30 上线这版,买一台腾讯云 Lighthouse 4C8G + 一套 TencentDB for PostgreSQL 就够了。**

---

## 一、市面上可选的服务器提供商

国内小程序后端能用的,主要就这几家:

| 提供商 | 轻量级产品 | 标准云服务器 | 托管 PostgreSQL |
|---|---|---|---|
| **腾讯云** | Lighthouse(轻量应用服务器) | CVM | TencentDB for PostgreSQL |
| 阿里云 | 轻量应用服务器 | ECS | RDS for PostgreSQL |
| 华为云 | 耀云服务器 | ECS | RDS for PostgreSQL |
| 火山引擎 | 轻量应用服务器 | ECS | 云数据库 PostgreSQL |

对我们来说,**首选腾讯云**,理由只有一条:**我们是微信小程序,和微信生态在同一家云上,备案、域名、审核这些麻烦事都最顺**。其他几家技术上都能跑,但会多出跨平台对接的小麻烦。

---

## 二、腾讯云里选哪个:Lighthouse 还是 CVM

### Lighthouse(轻量应用服务器)
- **像什么**:套餐式打包,开箱即用,适合"一台机器跑一个应用"的场景
- **优点**:便宜、上手快、套餐透明、运维压力小
- **缺点**:配置固定,不能灵活挂载各种企业级组件

### CVM(云服务器)
- **像什么**:标准云服务器,啥都能装、啥都能配
- **优点**:灵活、能做集群、能做负载均衡、能弹性扩容
- **缺点**:贵、配置项多、前期上手慢

---

## 三、我们应该怎么选

### 4/30 首发 —— 用 Lighthouse

- **应用服务器**:腾讯云 Lighthouse,**4 核 8G**
- **数据库**:TencentDB for PostgreSQL(**不和应用同机**,单独托管)
- **AI 中间件**:直接用 Dify 云端账号,不自建

**为什么是 Lighthouse 而不是 CVM**:首发就是一台机器跑 NestJS 后端,没有集群、没有负载均衡、没有弹性伸缩的需求,Lighthouse 的套餐价比 CVM 便宜一截,配置够用,上手还快。CVM 的灵活性现阶段全都用不上,纯属多花钱。

**为什么数据库要独立**:应用和数据库混在一台机上,出故障时一起死,备份和扩容也都更麻烦。TencentDB 自带备份和监控,省下的人力比省下的钱值钱。

### 用户量起来后(V1.1)—— Lighthouse 升配

- Lighthouse 从 4C8G 升到 8C16G
- 数据库加主从
- 图片迁到对象存储 COS

这一步只改部署,不动代码。

### 再往后(V2)—— 换 CVM 做集群

只有在下面这些情况出现时,才考虑从 Lighthouse 迁到 CVM:
- 单机撑不住并发
- 需要多实例 + 负载均衡
- 需要把后端拆成多个服务分开部署

**在这之前不要动。** Lighthouse 完全够一树 OPC 的前几千用户。

## 四、多少用户 × 多少钱

> 以下都是腾讯云公开价格的大致量级,实际以下单为准(买年付通常便宜 30% 左右)。
> 用户数按"**注册用户**"算,**日活(DAU)约为注册用户的 10% ~ 20%**。

### 4.0 一张总表(最重要的)

| 档位 | 注册用户 | 日活(DAU) | 月度总成本 | 何时升级 |
|---|---|---|---|---|
| **T0 内测** | 0 ~ 500 | 0 ~ 100 | **约 600 ~ 1000 / 月** | 月活破 500 或 API 费用连续两月超 500 |
| **T1 小规模** | 500 ~ 5000 | 100 ~ 1000 | **约 1500 ~ 3000 / 月** | DAU 破 1000 或数据库 CPU 常态 > 60% |
| **T2 中规模** | 5000 ~ 50000 | 1000 ~ 10000 | **约 4000 ~ 10000 / 月** | DAU 破 5000 或单机带宽撑不住 |
| **T3 集群** | 50000+ | 10000+ | **约 15000 ~ 30000+ / 月** | — |

**解读**:
- 每进一档,成本大概 **2~3 倍** 往上翻,但承载用户量是 **10 倍** 的跃迁
- 升档的信号不是"用户数到了",而是"服务器指标顶了"——CPU、带宽、数据库连接数
- 智谱 API 费用是唯一按**调用量**线性涨的,其他都是阶梯

---

### 4.1 T0 内测档:注册 500 人以内 / DAU ≤ 100

**对应阶段**:4/30 首发 → 邀请制内测

| 项目 | 配置 | 价格(元/月) |
|---|---|---|
| Lighthouse 应用服务器 | 4C8G / 8M 带宽 / 180G SSD | 150 ~ 200 |
| TencentDB for PostgreSQL | 基础版 2C4G / 100G | 300 ~ 400 |
| 域名 + HTTPS | .com + 免费 DV 证书 | 5 |
| Dify 云端 | 免费额度 | 0 |
| 智谱 GLM API | 按调用量 | 100 ~ 300 |
| **合计** | | **约 600 ~ 1000 / 月** |

**估算口径**:
- 100 个 DAU,每人每天平均跟 AI 对话 5~10 轮 ≈ 每天 500~1000 次模型调用
- 按智谱 GLM-5 + 少量 flash 混合均价算,每月 API 费用通常落在 100~300 元
- Lighthouse 4C8G 的带宽 8M,够这个量级 **10 倍都有余**,所以不是瓶颈

**每用户成本**:约 **1.2 ~ 2 元 / 注册用户 / 月**(内测期偏贵,因为固定成本摊不薄)

---

### 4.2 T1 小规模档:注册 500 ~ 5000 / DAU 100 ~ 1000

**对应阶段**:V1.0 公开放量,邀请制放开

| 项目 | 配置 | 价格(元/月) |
|---|---|---|
| Lighthouse 应用服务器 | 4C8G → 8C16G / 10M 带宽 | 400 ~ 500 |
| TencentDB for PostgreSQL | 基础版升到 4C8G 或 高可用 2C4G | 600 ~ 900 |
| COS 对象存储 | 图片 / 海报 | 20 ~ 50 |
| Dify 云端 | 付费套餐(Sandbox → Pro) | 0 ~ 400 |
| 智谱 GLM API | 按调用量 | 500 ~ 1200 |
| **合计** | | **约 1500 ~ 3000 / 月** |

**估算口径**:
- 1000 DAU × 5~10 轮/人 ≈ 每天 5000~10000 次调用
- 智谱 API 按量计费,这一档 API 费用是大头
- Dify 免费额度通常这一档会撑不住,升付费

**每用户成本**:约 **0.3 ~ 0.6 元 / 注册用户 / 月**(已经开始摊薄)

---

### 4.3 T2 中规模档:注册 5000 ~ 50000 / DAU 1000 ~ 10000

**对应阶段**:V1.1 稳定放量期

| 项目 | 配置 | 价格(元/月) |
|---|---|---|
| Lighthouse 或 CVM | 8C16G 单机 或 CVM 4C8G × 2 + 负载均衡 | 800 ~ 1500 |
| TencentDB for PostgreSQL | 高可用版 4C8G(主从)+ 200G | 1200 ~ 2000 |
| Redis 缓存 | 1G 基础版 | 100 ~ 200 |
| COS 对象存储 | 按用量 | 100 ~ 300 |
| Dify 云端 | Pro 套餐 | 400 ~ 800 |
| 智谱 GLM API | 按调用量 | 1500 ~ 5000 |
| **合计** | | **约 4000 ~ 10000 / 月** |

**估算口径**:
- 10000 DAU × 5~10 轮 ≈ 每天 5~10 万次调用
- 这一档 **API 费用开始成为主导**,占总成本的 40~50%
- 这时候应该开始考虑:**把低价值标准任务(字段抽取、固定输出)切到 GLM-4-flash**,API 费用能砍一半
- 数据库此时也得升到高可用,否则一次宕机可能丢几小时数据

**每用户成本**:约 **0.1 ~ 0.3 元 / 注册用户 / 月**(成本结构已经健康)

---

### 4.4 T3 集群档:注册 50000+ / DAU 10000+

**对应阶段**:V2 规模化

| 项目 | 配置 | 价格(元/月) |
|---|---|---|
| CVM 集群 + 负载均衡 | CVM 8C16G × 3 + CLB | 3000 ~ 5000 |
| TencentDB for PostgreSQL | 高可用 8C16G + 只读实例 | 3000 ~ 6000 |
| Redis 高可用 | 4G 主从 | 500 ~ 1000 |
| COS + CDN | 图片加速 | 500 ~ 1500 |
| Dify 或自建 | 企业版或自建 Dify 集群 | 1000 ~ 3000 |
| 智谱 GLM API | 按调用量(已做分层降本) | 5000 ~ 15000 |
| 监控 / 日志 / 告警 | CLS + 云监控 | 300 ~ 800 |
| **合计** | | **约 15000 ~ 30000+ / 月** |

**估算口径**:
- 到这一档,**成本不再是问题,稳定性才是**。运维压力和开发压力都会陡增
- 这时如果还没做"小模型微调 + 旗舰模型混合",API 账单会失控
- 这一档之前最重要的动作是**建立成本监控看板**,不然月底账单会吓人

**每用户成本**:约 **0.3 ~ 0.6 元 / 注册用户 / 月**(因为引入了高可用冗余,单位成本会略回升,但体验也更稳)

---

### 4.5 几个最容易被忽略的隐藏成本

1. **微信认证费**:每年 **300 元**,小程序上架必交
2. **ICP 备案**:免费,但耗时 7~20 天,**必须提前办**
3. **短信服务**(如果上了手机号登录):约 **0.04 元/条**
4. **API 费用的"反向冲击"**:用户在 AI 对话里连续追问 10 轮以上,单次会话可能烧掉几毛钱——这是 AI 类小程序独有的成本风险

---

### 4.6 一句话

> **首发一个月 1000 块以内(500 用户),放量后每月 1500~3000(5000 用户),再往上每月 4000~10000(5 万用户),集群档才破万。每个注册用户的月摊成本能稳定在 0.1~0.6 元区间。**

## 五、模型微调:什么时候做、去哪里做

### 5.1 先把时间点说清楚

**4/30 首发不做微调**。原因只有一个:**没有真实用户数据**。上线前拿内部构造的语料去微调,等于把假设固化进权重,上线后反而改不动。

真正考虑微调,应该是 **上线后 1~2 个月**(T1 档,约 5000 注册用户)这个时间点:

- 这时资产盘点、月度体检、日常任务已经跑出真实对话
- 能看出"哪些任务 Prompt 调不动"、"哪些任务高频重复"
- 有足够的数据量可以支持 LoRA(几百到几千条高质量样本就够了)

### 5.2 为什么必须选"国内的线上方案"

自建 GPU 不在候选清单里,理由很直接:

1. **一张 A100 80G 显卡报价就是十几万**,加上电费、机房、运维,不是一个小程序项目该碰的东西
2. **微调只是"阶段性动作"**,训练完就闲置了,GPU 空转浪费
3. **推理部署还要再占一张卡**,又是持续支出
4. **国外方案(OpenAI、Together、Replicate)有合规风险**,不能用来处理中国用户数据

所以结论是:**把 GPU 这件事完全外包给国内云厂商的"微调平台",我们只管上传数据、选基座、下载权重或直接调 API。**

### 5.3 国内可选的"在线微调"平台

| 平台 | 所属 | 支持的模型 | 微调方式 | 特点 |
|---|---|---|---|---|
| **智谱 BigModel 微调平台** | 智谱 AI | GLM-4 系列(flash / air / 9b 等) | LoRA / 全参 | **和我们现在用的 GLM 生态一致,首选** |
| 阿里百炼(DashScope) | 阿里云 | 通义千问 Qwen 系列 | LoRA / SFT / DPO | 生态成熟,Qwen 效果好 |
| 火山方舟 | 字节跳动 | 豆包 Doubao 系列 + 开源模型 | LoRA / SFT | 价格激进,推理便宜 |
| 百度千帆 | 百度 | 文心 ERNIE 系列 + 开源模型 | LoRA / SFT / RLHF | 工具链完整,适合企业 |
| 腾讯云 TI 平台 | 腾讯云 | 混元系列 + 开源模型 | LoRA / SFT | 和我们服务器同生态,省事 |

**这些平台的共同优点**:
- 不用自己管 GPU,按"训练时长 + 推理 token 数"计费
- 训完的模型可以直接生成一个私有 API Endpoint,像调普通 API 一样接入
- 数据合规由厂商背书(国内数据不出境)

### 5.4 我们应该选哪家

**首选:智谱 BigModel 微调平台**

理由:

1. **基座模型统一**。我们现在主模型就是 GLM-5,微调小模型也用 GLM-4.x,一个账号、一套 API Key,开发和运维都省事
2. **Dify 原生兼容**。微调后的模型可以直接在 Dify 工作流里替换节点模型,不用改编排逻辑
3. **数据流转最短**。数据从 PostgreSQL 导出 → 智谱平台 → 部署回 API,全程在一个厂商内
4. **和现在的计费账户合一**,老板只要看一张账单

**备选:火山方舟(豆包)**

只在以下情况考虑:
- 发现 GLM 小模型在我们场景下就是调不动
- 或者 API 费用成为主要成本瓶颈,需要换更便宜的推理

其他几家(百炼 / 千帆 / 腾讯 TI)技术上都能用,但切换成本 > 收益,除非有明确理由,不建议动。

### 5.5 微调大概要花多少钱

> 以智谱 BigModel 公开的大致费率为参考,实际以平台实时报价为准。

**一次性训练成本**:

| 基座模型 | 微调方式 | 训练样本 | 训练时长 | 大概费用 |
|---|---|---|---|---|
| glm-4-flash | LoRA | 1000 条 | 1~2 小时 | **几十到一百多元** |
| glm-4-air | LoRA | 3000 条 | 2~4 小时 | **两三百元** |
| glm-4.5-air | LoRA | 5000 条 | 4~8 小时 | **五六百到一千元** |
| glm-4.5 | LoRA | 10000 条 | 8 小时 + | **一千到三千元** |

**一次微调跑下来(含失败重跑 2~3 次),总预算大概 500 ~ 3000 元。** 这比自租 GPU 便宜一个数量级,也比自建省掉了全部固定资产投入。

**持续推理成本**:

- 微调后的模型推理费率通常比对应的基础模型**贵 20% ~ 50%**
- 举例:glm-4-flash 每百万 token 约 1 元,微调后约 1.2~1.5 元
- 但微调能让你用小模型干掉原本要走 GLM-5 的任务,**综合成本反而下降**

### 5.6 推荐落地路线(和用户档位对齐)

| 档位 | 用户数 | 微调策略 |
|---|---|---|
| **T0 内测** | 0 ~ 500 | **完全不微调**,全部走 Prompt + Dify |
| **T1 小规模** | 500 ~ 5000 | 开始收集高频场景语料,做第一次"字段抽取 + 固定格式输出"的小模型 LoRA(glm-4-flash) |
| **T2 中规模** | 5000 ~ 50000 | 铺开小模型分层:客服话术、月度体检结构段、资产分类各训一个 LoRA;主模型仍用 GLM-5 |
| **T3 集群** | 50000+ | 考虑定制一个"行业基座"(glm-4.5 级别),或走智谱官方的"企业定制通道" |

### 5.7 什么时候才考虑微调 GLM-5 / GLM-5.1 本身

即便到 T3,我们也 **不自己微调 GLM-5 / GLM-5.1**,而是:

- 找智谱商务走**官方企业定制通道**(他们内部有算力,按项目报价)
- 我们只提供数据 + 验收标准,拿回 API 调用方式
- 单次报价通常在 **几万到几十万**,只在有明确 ROI 时才做

**这件事至少是 V2 之后的讨论,写在这里只是为了让老板知道"有这条路"。**

### 5.8 一句话

> **上线三个月内不微调;之后微调全部走国内云平台(首选智谱 BigModel),一次训练预算 500~3000 元;GPU 这件事我们不碰,永远不碰。**

---

## 六、一句话总结

> **4/30 上线这版,腾讯云 Lighthouse 4C8G + TencentDB for PostgreSQL 独立托管 + Dify 云端,月度预算 1000 元以内。用户起量后按档升配,微调走智谱 BigModel 线上平台,永远不自建 GPU。**
