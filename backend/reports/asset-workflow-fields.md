## 6. 一句话总结

我们真正想从用户身上问出来的核心字段，其实就是 3 组：

- `profileSnapshot`：用户到底有哪些资产
- `dimensionReports`：这些资产分别怎么判断
- `reportBrief`：综合结论和报告收口摘要

其余很多字段，都是为了让系统“继续问、不中断、能生成报告”而存在的控制字段。

## 7. `profileSnapshot` 字段对应想收集的证据

这一层更像“系统想沉淀的用户画像栏目”。每个栏目都不是靠用户一次性自报得到的，而是通过多轮追问逐步补齐。

| 中文栏目 | 想收集的证据 | 示例问题 |
| --- | --- | --- |
| `真实案例` | 至少 1-2 个真实发生过的事件，包含时间、场景、动作、结果 | “过去一年里，哪件事是你亲手推进、最后拿到具体结果的？” |
| `热爱与价值观` | 用户真正愿意长期投入的方向、在意的结果、排斥的工作方式 | “你最想长期做下去的事情是什么？为什么它对你重要？” |
| `能力资产` | 用户反复表现出的动作能力、解决问题的方法、可迁移做法 | “这件事里你亲手做了哪一步？最后为什么能成？” |
| `资源资产` | 已有工具链、渠道、权限、内容池、人脉入口、可调动资源 | “你现在手里已经能直接调动的资源有哪些？” |
| `认知资产` | 用户的独特判断、行业理解、方法论、信息差、取舍原则 | “你对这个方向最不一样的判断是什么？” |
| `关系资产` | 愿意信任、帮忙、转介绍、继续聊、追问价格的人或圈层 | “最近一次真的愿意继续聊、愿意给反馈或追问价格的人是谁？” |
| `四圈线索` | 能力、资源、认知、关系四类资产之间如何互相放大 | “如果把你的能力、资源和关系放在一起，哪一条最可能先形成闭环？” |
| `优势分型线索` | 用户更像哪种优势型人格或工作风格，例如执行型、连接型、判断型、结构型 | “你更擅长把事情推起来，还是把复杂问题看透、讲清楚？” |
| `跨界复利线索` | 不同经历、行业、技能之间能形成复利的组合点 | “你过去哪些看起来不相关的经历，放在一起反而形成了独特优势？” |
| `内部判断` | 系统基于用户证据做出的阶段性判断、待验证点、不确定性 | 这部分不是直接问出来的，而是系统根据前面证据整理出来的判断 |

## 8. `dimensionReports` 每个小报告真正想回答什么

这一层不是简单复述用户的话，而是把各维度证据压缩成可判断、可继续推进的小报告。

统一结构通常包括：

- 已识别资产
- 证据案例 / 可调用资源 / 独特判断 / 信任网络
- 可迁移性 / 稀缺性 / 组合优势 / 第一单帮助可能性
- 初步变现性
- 当前判断

### `热爱与价值观小报告`

想回答的问题：

- 用户到底真正在意什么
- 用户愿意长期投入什么
- 用户不适合什么样的工作或商业路径

常见追问：

- “你做哪类事情最不容易耗竭？”
- “什么样的结果会让你觉得这件事值得继续做？”

### `能力资产小报告`

想回答的问题：

- 用户真正会做成什么事
- 这种能力是不是可迁移、可复用、可交付
- 这是不是普通能力，还是更有杠杆的能力组合

常见追问：

- “这件事里你亲手做了哪一步？”
- “最后拿到了什么结果？”
- “这套做法能不能迁移到别的场景？”

### `资源资产小报告`

想回答的问题：

- 用户手里到底有什么现实资源
- 这些资源是否已经被调用过
- 这些资源是否离变现或验证足够近

常见追问：

- “你现在已经能直接调用哪些工具、权限、渠道或人？”
- “最近一次这套资源真的被用来解决问题，是什么时候？”
- “有没有导流、反馈、询价或高意向信号？”

### `认知资产小报告`

想回答的问题：

- 用户有没有独特判断，而不是泛泛观点
- 这些判断有没有进入真实决策或取舍
- 是否能形成方法论或跨界优势

常见追问：

- “你对这个方向最核心的判断是什么？”
- “你最近一次按这个判断做了什么取舍？”
- “这套判断能不能迁移到更多人或更多场景？”

### `关系资产小报告`

想回答的问题：

- 谁真的愿意信任用户、帮助用户、继续聊、转介绍
- 这些关系有没有进入实际动作，而不只是泛泛认识
- 是否存在前付费转化信号

常见追问：

- “最近一次是谁愿意继续聊或认真给你反馈？”
- “有没有人主动问过价格、交付方式、试用方式？”
- “这些关系里谁最可能带来第一单或第一个种子用户？”

## 9. `reportBrief` 最终想收口出的信息

`reportBrief` 不是再问一个新维度，而是把前面几层东西压缩成“可以生成正式报告”的摘要。

它真正想回答的是：

- 四维核心结论分别是什么
- 四圈交汇点在哪里
- 用户最值得放大的 3 项核心资产候选是什么
- 哪些是强可变现资产，哪些还只是潜力项
- 当前最大的误判或待验证点是什么

可以理解成：

- `profileSnapshot`：素材库
- `dimensionReports`：分维判断
- `reportBrief`：最终收口摘要

## 10. 一个更实用的理解方式

如果只用一句业务话来概括，系统其实想从用户身上问出 4 类证据：

1. `做成过什么`
2. `手里有什么`
3. `看懂了什么`
4. `谁愿意信你`

然后再把这 4 类证据整理成 3 层结果：

1. `profileSnapshot`：分栏沉淀
2. `dimensionReports`：分维判断
3. `reportBrief`：综合收口
