你负责跟我聊天，我负责帮你赚钱。

## 二、用户画像（五类核心用户）

### 画像一："迷茫探索者"
- **典型用户**: 职场3-8年白领，25-38岁，想做OPC但完全没方向
- **口头禅**: "我想创业，但不知道做什么"
- **潜意识驱动力**: 存在性焦虑 + 确认偏误 + 决策疲劳
- **核心恐惧**: "选错方向会浪费时间"
- **AI路由**: 小管家自动识别"迷茫"信号 → 启动 Asset Agent 资产盘点对话

### 画像二："行动瘫痪者"
- **典型用户**: 已有想法或领域认知，但始终停留在"想"的阶段
- **口头禅**: "我知道要做什么，但就是做不了"
- **潜意识驱动力**: 损失规避 + 冒名顶替综合征 + 意图-行动差距
- **核心恐惧**: "做了还失败 = 证明我真的不行"
- **AI路由**: 识别"拖延/完美主义"信号 → 切入 Mindset Agent 卡点拆解

### 画像三："单打独斗者"
- **典型用户**: 已在做自由职业/副业，有单但收入不稳定，月入5k-30k
- **口头禅**: "已经在做了，但收入不稳定"
- **潜意识驱动力**: 稀缺性恐慌 + 控制幻觉 + 沉没成本谬误
- **核心恐惧**: "停下来思考 = 错过眼前的单子"
- **AI路由**: 识别"已有业务但不稳定" → System Agent 稳定化模块

### 画像四："天花板撕裂者"
- **典型用户**: 月入稳定3-10万，但个人时间已被完全填满，30-45岁
- **口头禅**: "想用AI放大规模，但不知道从哪入手"
- **潜意识驱动力**: 时间稀缺性恐惧 + 工具负担过载 + 委托焦虑
- **核心恐惧**: "把业务交给AI会搞砸我的招牌"
- **AI路由**: 识别"已到达天花板" → Scaling Agent 规模引擎

### 画像五："合规求助者"
- **典型用户**: 业务跑通，面临财税、注册、团队合作等"现实世界"问题，30-50岁
- **口头禅**: "我需要专业的财税建议和合伙人匹配"
- **潜意识驱动力**: 专业权威依赖 + "被罚"的恐惧 + 决策心理超载
- **核心恐惧**: "不懂规则，万一踩坑怎么办"
- **AI路由**: 识别"合规类问题" → Ops & Tax Agent / Co-Founder Agent

## 三、AI角色系统（5个角色）

### 角色1: 一树
- **对应Agent**: Master Agent（主人格/默认状态）
- **颜色**: #0D0D0D（黑色）
- **图标**: 树+箭头的圆形icon
- **语气**: calm, grounded（沉稳、落地）
- **职责**: 日常闲聊、分流路由、每日任务推送、串联上下文、在各角色间无缝切换
- **示例对话**: "小明，说说你现在的状态？" / "好的，我先帮你盘一下。"

### 角色2: 一树 · 挖宝
- **对应Agent**: Asset Agent + Opportunity Agent（合并）
- **颜色**: #534AB7（紫色）
- **图标**: 放大镜+加号
- **语气**: curious, probing（好奇、挖掘式）
- **职责**: 发现用户能做什么。深度对话挖掘隐藏资产，生成商业方向，Mom Test验伪
- **输出卡片**: 资产雷达图、机会评分矩阵
- **示例对话**: "你之前工作里，什么事做起来最不费劲但别人觉得很厉害？"

### 角色3: 一树 · 搞钱
- **对应Agent**: Execution Agent
- **颜色**: #10A37F（绿色）
- **图标**: 准星/十字瞄准
- **语气**: direct, action-driven（直接、结果导向）
- **职责**: 帮用户赚到钱。产品化、三层定价、拿第一单、转化优化、触达话术生成
- **输出卡片**: 产品结构卡片、三层定价卡片、触达话术卡片
- **示例对话**: "产品不错，但你的定价有问题。客户买的是结果，不是你的时间。"

### 角色4: 一树 · 扎心
- **对应Agent**: Mindset Agent
- **颜色**: #E24B4A（红色）
- **图标**: 闪电
- **语气**: blunt, no-BS（毒舌、不废话）
- **职责**: 识别拖延、完美主义、信息上瘾等卡点，直接拆穿借口，生成48小时行动计划
- **输出卡片**: 警告卡片（红色边框）、48h行动计划+倒计时
- **示例对话**: "你不是没准备好，你是在逃避。" / "你已经3天没触达客户了。"

### 角色5: 一树 · 管家
- **对应Agent**: System Agent + Ops & Tax Agent + Scaling Agent（合并）
- **颜色**: #378ADD（蓝色）
- **图标**: 四宫格/仪表盘
- **语气**: reliable, structured（靠谱、条理清晰）
- **职责**: 收入稳定化、Profit First财务分配、园区政策匹配、税务筹划、代发薪资、AI自动化放大
- **输出卡片**: 账户分配饼图、园区政策匹配卡片、收入模型卡片
- **示例对话**: "今天是资金分配日。本月收入18,000元，建议分配如下...确认？"

### 角色切换规则
- 用户在任何对话中随时可以问任何问题
- Master Agent自动判断意图并切换到对应角色
- 切换时header中间的图标颜色和角色名同时变化（0.3秒渐变过渡）
- 用户无需任何手动操作
- 对话流不中断，用户感知是"同一个人切换了模式"

## 四、屏幕架构

### Header三点式布局（顶部固定栏）
- **左侧**: 用户头像（点击滑出左侧边栏）
- **中间**: 当前角色图标+名称（动态切换，如"一树OPC" / "一树·搞钱"）
- **右侧**: 小树icon（点击打开"我的一树"成就系统全屏页）

### 主屏幕
- 对话界面占据85%屏幕空间
- 底部输入栏: "+"按钮（项目弹窗）+ 输入框 + 语音按钮

### 左侧边栏（从左滑出或点击头像触发）
- **第一区 - 用户卡片**: 头像+昵称。可点击进入"我的档案"页面
- **第二区 - + 新对话**: 醒目的黑色按钮
- **第三区 - 项目列表（PROJECTS）**: 活跃项目列表，每项显示项目名+状态+待办数
- **第四区 - 工具（TOOLS）**: 三个并排按钮——「AI助手」「IP助手」「公司」
- **第五区 - 对话历史（RECENT CHATS）**: 按时间倒序，每条显示AI自动生成的主题标题
- **第六区 - 底部**: 设置 + 帮助

### "+"按钮弹窗（底部半屏）
- 项目列表，每个项目卡片显示: 项目名+当前阶段+待办数
- 底部"+ 新项目"入口

## 五、完整交互流程（25个屏幕）

### 第一阶段: 首次进入（Screen 1-7）

**Screen 1: 首页（Landing）**
- 内容: 一树OPC / 你的一人公司搭子 / 你负责跟我聊天，我负责帮你赚钱。/ CTA「跟一树聊聊？」/ 底部"2,847位一人公司创业者正在这里搞钱"
- 设计: 大量留白，黑色CTA按钮，无多余装饰

**Screen 2: 一树打招呼 + 登录卡片（Step 1a）**
- 点击CTA后直接进入对话界面（非独立登录页）
- 一树消息气泡: "嘿，我是一树。在开始之前，先让我认识一下你。"
- 紧接着对话流中出现登录卡片: 微信一键登录按钮 + 用户协议链接
- 关键: 登录是对话的一部分，不是对话的前置障碍

**Screen 3: 登录成功，卡片原地更新（Step 1b）**
- 用户点击微信登录后，登录卡片原地更新为: 用户头像+昵称+"已登录"
- 一树开始打字（显示typing动画）

**Screen 4: 确认昵称（Step 2a）**
- 一树消息: "[微信昵称]，我怎么称呼你比较好？"
- 快捷回复: 「就叫[昵称]」「叫我别的名字」

**Screen 5: 用户选完昵称，一树打字中（Step 2b）**
- 用户选择以消息气泡形式发出
- 一树typing动画

**Screen 6: 分流问题 + 园区钩子卡片（Step 3a）**
- 一树消息: "好的[昵称]，说说你现在的状态？"
- 快捷回复（3个）: 「想做一人公司，没方向」「有想法，迈不出第一步」「在做了，想用AI放大规模」
- 紧接着一树自动多发一张钩子卡片:
  - 标题: "你知道吗？"
  - 正文: "全国有不少园区正在抢一人公司入驻，免费注册地址、税收返还、甚至直接发钱。你可能符合好几个，但根本没听说过。"
  - CTA: 「帮我查查能薅什么」
- 两条路径互不干扰: 用户可先选状态进主流程，也可先点园区钩子

**Screen 7: 四条分支路径（Step 4 path A/B/C/D）**
- **Path A（没方向）**: 一树说"没关系，大多数人一开始都没方向。我先帮你盘一下——你之前做什么工作？" + 快捷回复
- **Path B（迈不出）**: 一树说"有想法但动不了，这事儿太常见了。先聊聊——你的想法大概是什么方向？" + 快捷回复
- **Path C（放大规模）**: 一树说"已经在赚钱了，不错。我先了解一下——你现在靠什么在赚钱？" + 快捷回复
- **Path D（查园区）**: 一树说"好眼光，很多人不知道这些政策白白错过了。我先问你几个问题——你现在注册公司了吗？" + 快捷回复

### 第四阶段: 日常循环（Screen 19-20）

**Screen 19: 每日任务 + 完成反馈闭环**
- **任务推送**: 用户每天首次打开，一树自动发送"今日任务"卡片（最多3条），每条带勾选框+关联项目标签
- **完成反馈**: 用户勾选完成后，一树自动追问"结果怎么样？你想聊聊吗？"（开放式追问，非预设选项）
- **策略调整**: 一树根据用户的自由回复分析情况，给出具体的下一步建议
- 示例闭环: 完成"触达5个客户" → 一树追问结果 → 用户说"有2个感兴趣但犹豫" → 一树给出针对性跟进话术

- 分享图文案风格: 有获得感但不炫耀（如"原来我的隐藏资产比我想的多得多"）

## 六、里程碑体系（15个节点）

### 播种期（对应 Asset + Opportunity Agent）
1. **完成资产盘点** — 生成第一张资产雷达图
2. **锁定商业方向** — 从3个候选方向中选定1个
3. **完成客户验证** — 按Mom Test完成5次真实客户对话
4. **通过Go/No-Go** — 机会评分达到20分以上

### 发芽期（对应 Execution Agent）
5. **拿下第一单** — 完成首次真实交付并收款
6. **完成产品化** — 提炼出标准服务包
7. **建立三层定价** — 入门/核心/高端定价体系定型

### 扎根期（对应 System Agent）
8. **月收入连续稳定** — 连续3个月有收入进账
9. **启动Profit First** — 开设五账户并完成第一次资金分配
10. **建立获客飞轮** — 有一条可重复获客渠道持续运转

### 成林期（对应 Scaling + Ops Agent）
11. **完成园区入驻** — 拿到营业执照或虚拟注册地址
12. **搭建第一条自动化工作流** — 用AI替代一个重复性环节
13. **月入过万**
14. **月入过十万**
15. **种出第二棵树** — 启动第二个OPC项目（解锁"一树成林"场景）

## 八、视觉设计规范

### 色彩体系
| 角色 | 色值 | 使用场景 |
|------|------|----------|
| 对话背景 | #F7F7F8 | 主屏背景 |
| AI消息气泡 | #FFFFFF | 一树发出的消息 |
| 用户消息气泡 | #0D0D0D | 用户发出的消息 |
| 强调色(CTA) | #10A37F | 行动按钮、完成状态、正向反馈 |
| 警告色 | #EF4444 | 扎心卡片、紧急提醒 |
| 侧边栏背景 | #FAFAFA | 左侧滑出侧边栏 |
| 快捷按钮边框 | #D0D5DD | 快捷回复按钮默认边框 |

### 色彩规则
90%黑白灰为主。颜色只在行动状态时出现，不做装饰用途。

### 字体规范
| 元素 | 字号 | 字重 |
|------|------|------|
| 品牌名 | 28px | Medium(500) |
| 页面标题 | 22px | Medium(500) |
| 正文 | 15px | Regular(400) |
| 辅助文字/时间戳 | 13px | Regular(400) |
| 按钮文字 | 15px | Medium(500) |
| 快捷回复 | 14px | Medium(500) |

### 字体选择
系统字体: -apple-system, BlinkMacSystemFont, "Helvetica Neue", "PingFang SC"
仅使用两种字重: 400 Regular, 500 Medium。永远不用Bold，永远不用Light。

### 间距与密度
- 对话区: 大量留白，气泡间距12px，营造"朋友聊天"的舒适感
- 卡片内部: 紧凑密集，内边距16px，每个像素都有信息价值
- 两者形成有意的反差: 闲聊宽松 + 数据紧凑

## 九、快捷回复设计规范

### 三种类型
| 类型 | 触发场景 | 示例 |
|------|----------|------|
| 选择型 | AI提出封闭式问题 | 「技术/编程」「营销/运营」「设计/创意」「其他」 |
| 确认型 | AI给出建议需用户确认 | 「好的，继续」「我想换个方向」 |
| 行动型 | AI要求用户执行动作 | 「开始执行」「记录结果」 |

### 设计规范
- 数量: 每次最多4个
- 文案长度: 每个按钮最多8个字
- 布局: 横向滑动排列，不占用对话区垂直空间
- 触摸目标: 最小44px高度
- 样式: 圆角胶囊(border-radius:20px)，14px字号，0.5px边框
- "其他"选项: 当问题有开放性时，最后一个按钮为"其他→"，点击后弹出输入框

## 十、微信小程序技术约束

| 约束项 | 设计应对 |
|--------|----------|
| 包体积 ≤ 20MB | 图片统一WebP，动画用Lottie，重资源CDN加载 |
| 首屏加载 ≤ 1.5s | 首屏仅加载对话界面+最新一条消息，侧边栏懒加载 |
| 输入体验 | 大量使用快捷回复减少手动输入，支持微信语音输入 |
| AI响应 | SSE流式传输，逐字展现。卡片在文字流完后以淡入动画出现 |
| 侧边栏 | 从左侧边缘右滑或点击头像触发，加载快速且无页面跳转 |
| 卡片渲染 | 使用小程序原生组件(Canvas/WebView)渲染图表卡片 |
| 模板消息 | 早8:00推送"今日任务"，晚21:00推送"今日复盘"。点击直达对应对话位置 |

## 十一、留存机制（按优先级排序）

### MVP阶段（P0）
1. **每日任务推送**: 早上一树主动发送今日任务卡片(≤3条)
2. **任务完成反馈闭环**: 勾选完成后一树追问"结果怎么样？你想聊聊吗？"
3. **里程碑解锁仪式**: 完成关键里程碑时的庆祝卡片+分享

### 增长阶段（P1）
4. **周报卡片**: 每周日推送，数据四宫格+一树点评+分享功能
5. **同路人社会证明**: 在用户停滞时推送同阶段用户的进展数据
6. **成果分享裂变**: 每张成果卡片可生成朋友圈分享图（带小程序码）

### 规模阶段（P2）
7. **月度商业体检**: 每月1号一树·管家自动发起系统性回顾
8. **一树日报灵感**: 每天推送一条跟用户阶段相关的灵感（案例/语录/市场机会）

## 十三、设计检查清单（前端开发每个决策点必须回答）

1. **这个功能是否可以在对话流中完成？** 如果需要跳转新页面，思考能否用嵌入式卡片或半屏弹窗替代。
2. **屏幕上的核心信息是否超过3条？** 如果超过了，删减。
3. **这个输出是否有明确的CTA按钮？** 如果只是信息展示而不引导行动，重新设计。
4. **用户是否能感受到进步？** 完成动画、阶段更新、进度反馈是否到位。
5. **这个设计是否让产品感觉像"一个懂你的人"而不是"一个工具"？** 一树的语气是否有温度，是否主动关心用户而不是被动等待指令。
