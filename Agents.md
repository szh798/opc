# AGENTS.md

## Project
这是「一树OPC」微信小程序原生前端项目。
产品定位：AI-Native 一人公司智能体操作系统
核心理念：对话即操作系统（Conversation as OS）

## Source of truth
1. PRODUCT_MANUAL.md 是业务和交互真相
2. screen_01.png ~ screen_25.png 是视觉真相
3. 若冲突：
   - 视觉以截图为准
   - 业务逻辑以 PRODUCT_MANUAL.md 为准

## Hard constraints
- 必须使用微信小程序原生
- 不要使用 uni-app / Taro / React / H5
- 以对话页为产品核心
- 其他能力采用侧边栏、半屏弹层、全屏页方式承载
- 页面代码中不要散落接口调用
- 所有数据请求走 services
- 支持 mock / real api 切换
- chat service 必须预留流式能力

## UI style
- 沉稳、锋利、有温度、不废话、像个活人
- 不要做成传统后台
- 90% 黑白灰
- 颜色只承担状态作用
- 对话区留白大
- 卡片内部信息密度高
- 快捷回复使用胶囊按钮
- Header 中间显示当前角色图标和名称
- 角色色：
  - 一树OPC：黑
  - 一树·挖宝：紫
  - 一树·搞钱：绿
  - 一树·扎心：红
  - 一树·管家：蓝

## 成长树页面规则

### 以谁为准
- `PRODUCT_MANUAL.md` 中的 Screen 21 是「我的一树 / 成长树页面」的业务真相。
- 我提供的树线稿图片是视觉参考。
- 如果两者有冲突：
  - 交互方式与页面结构以 `PRODUCT_MANUAL.md` 为准
  - 树的视觉风格以参考图为准

### 触发方式
- 成长树页面必须通过主聊天页**顶部固定栏区域下拉**触发打开。
- 不要实现成普通的下拉刷新。
- 不要继续把原来的右上角 icon 点击作为主入口。
- 不要实现成小弹窗。
- 整体感觉应该像一个被“拉出来”的全屏成就系统。

### 页面形态
- 这是一个**全屏页面**或**全屏覆盖层**。
- 顶部区域必须包含：
  - 左侧：关闭按钮 `X`
  - 中间：标题 `我的一树`
  - 右侧：当前阶段标签
- 顶部下方展示副标题：
  - 格式：`你的一树已成长到第X阶段 · [阶段名]`

### 核心视觉
- 页面主体必须包含一棵**居中的竖向成长树可视化**。
- 树的风格应该克制、产品化、线稿化：
  - 树干和分支：浅灰色
  - 叶子：浅绿色
  - 背景：干净、浅色、极简
- 不要做成卡通风。
- 不要做成游戏成就页风格。
- 树是这个页面最核心的视觉焦点。

### 动画规则
- 打开页面时不能生硬跳出。
- 页面需要“逐步显现”的感觉：
  1. 先出现全屏容器
  2. 树干和分支从下往上逐步显现
  3. 叶子逐步出现
  4. 里程碑节点和文案依次出现
- 动效应该轻、稳、克制。
- 不要使用夸张、炫技、花哨的动画。

### 里程碑时间线
- 成长系统必须包含一条**垂直时间线**。
- 里程碑必须有 3 种明确状态：

1. 已完成
   - 绿色实心圆
   - 勾号
   - 里程碑名称
   - 日期
   - 叶子数

2. 进行中
   - 橙色圆点
   - 当前指示
   - 里程碑名称
   - `进行中...`

3. 未解锁
   - 灰色空心圆
   - 里程碑名称
   - 整体透明度降低

- 时间线和树之间必须有视觉关联，不能像两个硬拼起来的独立模块。

### 叶子交互
- 已点亮或已解锁的叶子可以点击。
- 点击叶子后，打开该里程碑对应的成果卡片 / 产出物卡片。
- 结果弹层目前可以先用 mock 数据。
- 弹出的成果卡片风格应尽量复用项目中已有的 `result-card` 风格。

### 工程规则
- 项目必须保持为**微信小程序原生**实现。
- 优先采用可复用组件。
- 建议拆分为以下组件：
  - `growth-tree-page`
  - `growth-tree-visual`
  - `milestone-timeline`
  - `milestone-result-popup`
- 不要把全部逻辑和样式都塞进一个文件里。
- 先使用 mock 数据实现，但数据结构必须清晰。

### 建议的 mock 数据结构
- `currentStage`
- `currentStageLabel`
- `milestoneList`
- `unlockedLeaves`
- `selectedMilestone`
- `resultCardData`

每个里程碑建议至少包含：
- `id`
- `title`
- `status`
- `date`
- `leafCount`
- `resultCard`

### 禁止事项
- 不要实现成默认下拉刷新。
- 不要实现成小弹窗。
- 不要继续把旧的 icon 点击作为主触发方式。
- 不要只放一张静态树图而没有里程碑状态与交互。
- 不要把页面做得很吵、很花。

### 完成标准
只有满足以下条件，才算这个功能完成：

1. 主聊天页可以通过顶部固定栏下拉打开成长树全屏页
2. 成长树页可以通过 `X` 关闭并返回聊天页
3. 树有逐步显现的动画
4. `done / doing / locked` 三种里程碑状态视觉区分明确
5. 点击叶子可以打开成果卡片
6. 代码结构清晰、组件化、可维护

## 成长树实现偏好

- 优先选择稳定、可维护的实现方案，而不是炫技方案。
- 如果 canvas 太重，就优先用分层图片 + 遮罩 / 分段 reveal 动画来实现。
- 先把交互和结构做对，再逐步优化树的动画和细节。
- 如果有视觉细节缺失，先用最小一致性方案补足，并记录到 `TODO.md`。
