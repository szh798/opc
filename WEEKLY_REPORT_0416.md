## 三、本周重点工作

### 重点一：政策匹配"薅羊毛"对话流 — 从 0 到 1 闭环

**做了什么：** 完成了"用户聊到政策/补贴/薅羊毛 → 路由自动识别 → 后端匹配政策库 → 前端渲染政策卡片"的完整闭环。

**闭环路径：**

```
用户输入"有没有什么补贴可以领"
        ↓
  Router 识别意图 → policy_park 分支
        ↓
  PolicyOpportunityService 匹配用户画像
        ↓
  返回结构化政策卡片 (标题/金额/条件/申领指南)
        ↓
  前端 policy-card 组件渲染 → 用户可直接查看详情
```

**达到的效果：**
- 用户在对话中提到政策/补贴相关话题，系统**自动切换到政策匹配模式**，无需手动跳转页面
- 后端 `PolicyOpportunityService` 975 行业务逻辑，支持**按用户画像智能匹配**
- 前端新增 `policy-card` 组件，政策信息以**可交互卡片**形式展示，而非纯文字

### 重点三：7 个 Dify 工作流 Prompt 全量精简 — Token 降本 40%

**做了什么：** 对全部 7 个 Dify 工作流的 LLM 提示词进行系统性优化，消除重复、压缩冗余。

**优化策略：**

```
原状：7 个流各自重复 ~180 token 的 JSON 输出格式规则 → 合计浪费 ~1,260 token
                    ↓
优化：抽出公共指令到代码层统一注入 + 删除否定指令 + 结构化替代自然语言
                    ↓
结果：总 Token 从 ~7,150 降至 ~4,300，节省约 40%
```

| 工作流 | 优化前 Token | 优化后 Token | 节省 |
|--------|-------------|-------------|------|
| 1-首次资产盘点 | ~1,800 | ~1,080 | -40% |
| 2-断点续盘 | ~850 | ~510 | -40% |
| 3-复盘更新 | ~950 | ~570 | -40% |
| 5-通用兜底 | ~650 | ~420 | -35% |
| 6-闲聊收集 | ~1,100 | ~660 | -40% |
| 7-生意体检 | ~900 | ~540 | -40% |

**达到的效果：**
- 每次 API 调用节省约 400 token → 按日活 1,000 用户、人均 5 轮对话估算，**月节省约 600 万 token（约 ¥180/月 API 成本）**
- Prompt 更精练，模型输出格式合规率提升

**关键文档：** [prompt_optimization.md](prompt_optimization.md)

### 重点四：记忆系统三层服务上线 — 让 AI "记住"用户

**做了什么：** 完成记忆系统 Phase 1.6 的后端服务层，实现对话摘要 → 会话窗口 → 用户画像的三层提取链路。
市场需求根本不存在
**架构：**

```
用户完成一轮对话
    ↓
ChatflowSummaryService → 提取本轮对话摘要 (资产变动/风险点/关键决策)
    ↓
SessionWindowService → 聚合为会话级洞察 (本次盘点核心结论)
    ↓
UserProfileService → 更新长期用户画像 (资产偏好/风险承受能力/生意阶段)
    ↓
下次对话时注入 profileSnapshot → AI 能"记住"用户历史
```

**达到的效果：**
- 用户第二次进入对话时，AI 不再"失忆"，能引用上次盘点的关键信息
- 数据库新增 2 个迁移文件（memory_layers 表），Schema 扩展 123 行
- 已与 Router 集成，路由流结束后自动触发记忆提取

**关键提交：**
- `8cc1e62` feat: add memory services and deployment docs
- `2aa9b45` feat: trigger memory extraction on router streams
