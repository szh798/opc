# 业务规则记忆

## 资产盘点是当前最重要的业务方法论之一

它不是泛泛聊天，而是一套有明确判断标准、阶段机和输出结构的规则系统。

## 资产定义的核心判断

个人资产不是收入、兴趣或努力本身，而是“可持续创造价值的能力结构”。

一个要素要成为核心资产，至少满足下面 4 项里的 2 项：

- 可重复使用
- 被市场需求验证
- 可转化为收入
- 不完全依赖时间

默认不要把下面这些直接判成核心资产：

- 兴趣
- 学历证书
- 通用基础能力
- 只体现努力、没有市场信号的内容

## 四大资产维度

- 能力资产
- 资源资产
- 认知资产
- 关系资产

高杠杆资产的默认优先级：

- 认知资产
- 资源资产
- 关系资产
- 技能资产

## 盘点对话的阶段机

- `opening`
- `ability`
- `resource`
- `cognition`
- `relationship`
- `ready_for_report`
- `correction_loop`
- `report_generated`

## 盘点时必须遵守的对话规则

- 只问真实行为，不问抽象自评
- 从具体到抽象：先案例，再能力，再结果，再市场
- 每轮只问 1 个问题
- 当前阶段只追当前最关键的维度
- 用户模糊时，优先追问时间、场景、动作、结果
- 用户发散时，强制收敛到 3 个以内的核心能力或方向

## 可变现能力的默认判断

强可变现能力通常至少满足以下 5 项中的 3 项：

- 有明确用户群体
- 用户有痛点或损失
- 市场已有付费行为
- 能提供结果而非纯过程
- 不容易被替代

只满足 1 到 2 项时，通常只能算潜力能力。

## 强纠偏机制

以下任一情况成立，就不应进入总结报告：

- 没有真实案例
- 四个维度里没有 2 个以上达到中等以上
- 内容只停留在兴趣、学习、努力、通用能力
- 没有“世界需要”或“别人愿意付费”的信号

默认动作：

- 明确指出问题，不做安慰式模糊表达
- 要求补真实案例
- 必要时建议缩小问题范围或调整目标客群

## 总结报告的稳定结构

最终报告应围绕资产本身，不混入太多临时状态信息。默认结构包括：

1. 资产总览
2. 四大资产维度画像
3. 四圈交汇点分析
4. 优势分型
5. Top 3 核心资产
6. 可变现能力排序
7. 初步变现路径
8. 暂时不建议做的方向
9. 强纠偏结论
10. 下一步建议

## 与 prompt / workflow 的关系

- 这些规则适合做 prompt 和 workflow 的长期基准
- 比起直接依赖原始访谈文档，当前更稳的方式是先蒸馏成规则，再喂给编排器和报告生成器
- 如果新会话要改资产盘点相关 prompt，优先先读这份规则，再看原始业务文档
