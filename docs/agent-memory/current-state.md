# 当前阶段记忆

更新时间：2026-04-21

这份文档保留“当前仍高价值”的阶段信息，但不追求保存每一条临时排期。

## 当前产品成熟度

- 项目已经超过“静态原型”阶段，属于“能跑的 MVP”
- 4 月底目标仍然是体验版上线
- 但如果按“商业化产品”的标准看，当前还缺合规、可观测性、数据工程和运营闭环四类硬能力

## 当前最重要的上线层目标

- 跑通微信登录、主对话、项目、报告、分享等真实链路
- 用真实 API 替代主要页面和场景里的假数据/静默 fallback
- 收口生产配置、协议文案、审核材料、体验版验收

## 当前最重要的产品和工程缺口

按优先级看，最值得持续记住的是这几类：

- 合规红线：
  - 微信内容安全接入
  - AIGC 标识
  - 生产内容审核与敏感内容防护
- 可观测性与成本控制：
  - 结构化日志
  - 错误聚合
  - Dify 调用用量追踪
  - 业务级配额
- 数据与质量工程：
  - 前端埋点
  - 漏斗指标
  - 单元测试
  - CI / feature flag / 灰度
- 运营闭环：
  - 客服入口
  - 反馈分类处理
  - admin/workbench 能力

## 当前适合当“近期待办记忆”的事项

- 发布前检查必须围绕 `LAUNCH_CHECKLIST.md`
- 生产环境配置和微信审核材料，常常比代码本身更容易卡住上线
- 如果时间不够，应该优先保证主链路、审核、合规和生产配置，而不是继续扩功能面

## 当前不应误判为“已经完成”的事项

- 项目虽然已有记忆系统，但商业化所需的运营与合规基础设施还不完整
- 有些计划文档写得很全，不代表代码已经全落地
- 旧蓝图里提到的完整多智能体路线和大量 chatflow，并不等于当前版本都已实现

## 对新会话最有用的工作假设

- 这是一个正在冲体验版上线、但仍需强收口的项目
- 任何新增工作都应先判断它属于：
  - 上线阻塞项
  - 商业化基础设施
  - 可延后优化项
- 当任务涉及日期、上线状态、审核状态或负责人分工时，不应只靠这份记忆，应该重新核对最新现状
